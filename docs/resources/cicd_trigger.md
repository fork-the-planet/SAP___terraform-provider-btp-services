# btpservice_cicd_trigger (Resource)

Manages a trigger for a CI/CD job in the SAP BTP CI/CD service. Currently only `timer` type triggers are supported.

## Example Usage

```terraform
resource "btpservice_cicd_trigger" "nightly" {
  job  = btpservice_cicd_repository.app.name
  type = "timer"

  timer = {
    branch = "main"
    cron   = "0 2 * * *"
  }
}

# Weekday morning trigger
resource "btpservice_cicd_trigger" "weekday_morning" {
  job  = "my-pipeline-job"
  type = "timer"

  timer = {
    branch = "develop"
    cron   = "0 9 * * 1-5"
  }
}
```

## Schema

### Required

- `job` (String) Name or ID of the CI/CD job this trigger belongs to. Changing this forces recreation.
- `type` (String) Trigger type. Currently the only supported value is `timer`. Changing this forces recreation.

### Optional

- `timer` (Attributes) Timer schedule configuration. Required when `type` is `timer`. (see [below for nested schema](#nestedatt--timer))

### Read-Only

- `id` (String) Unique identifier of the trigger (assigned by the API on creation).

### Nested Schema for `timer`

Optional:

- `branch` (String) Branch to build on the timer schedule.

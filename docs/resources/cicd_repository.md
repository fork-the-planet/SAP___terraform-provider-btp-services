# btpservice_cicd_repository (Resource)

Manages a repository resource in the SAP BTP CI/CD service.

## Example Usage

```terraform
resource "btpservice_cicd_repository" "example" {
  name      = "my-app-repository"
  clone_url = "https://github.com/example/my-app"
}

# Private repository with clone credential
resource "btpservice_cicd_repository" "private" {
  name               = "my-private-repo"
  clone_url          = "https://github.com/example/my-private-app"
  clone_credential_id = btpservice_cicd_credential_basic_auth.deploy_user.id
}

# Repository with webhook event receiver for automated builds
resource "btpservice_cicd_repository" "with_webhook" {
  name      = "my-webhook-repo"
  clone_url = "https://github.com/example/my-app"

  event_receiver {
    active                      = true
    scm_type                    = "GITHUB"
    webhook_token_credential_id = btpservice_cicd_credential_webhook_secret.webhook.id
  }
}
```

## Schema

### Required

- `clone_url` (String) URL used to clone the repository (e.g. `https://github.com/example/repo`). Max 255 characters.
- `name` (String) Name of the repository. Allowed characters: `[a-zA-Z0-9_-]`, max 64 characters.

### Optional

- `clone_credential_id` (String) ID of the credential used to authenticate against the Git server when cloning. Max 63 characters.
- `cloud_connector_credential_id` (String) ID of the credential containing the Cloud Connector Location ID used for cloning. Max 63 characters.
- `event_receiver` (Attributes) Event receiver configuration for triggering builds via webhooks from your source code management system. (see [below for nested schema](#nestedatt--event_receiver))

### Read-Only

- `id` (String) Unique identifier of the repository (assigned by the API on creation).

### Nested Schema for `event_receiver`

Required:

- `active` (Boolean) Whether event processing is enabled for this repository.
- `scm_type` (String) Source code manager type. Allowed values: `GITHUB`, `BITBUCKET_CLOUD`, `BITBUCKET`, `GITLAB`, `AZUREREPOS`.

Optional:

- `webhook_token_credential_id` (String) ID of the webhook secret credential for this event receiver. Required for all SCM types except `BITBUCKET_CLOUD`. Max 63 characters.

Read-Only:

- `webhook_id` (String) Immutable technical ID of the webhook, assigned by the API on creation.
